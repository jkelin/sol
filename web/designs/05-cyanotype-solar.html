Observations 02—05

A smaller runtime starts in the source.

The compiler sees the relationships before the browser has to. Solix turns authored intent into narrow, owned DOM work.

I.

State feels like JavaScript

Writable declarations become signals. Derived constants become computed values. Inside components, read and assign them normally—without a .value ritual.

II.

Updates follow dependencies

Static templates are cloned once. Effects track signal reads, so writes schedule only the DOM operations that depend on them—not a component-wide rerender.

III.

The hard parts belong together

Typed compile-time routes, schema-backed forms, transitions, context, Suspense, Await, and error boundaries share the same ownership model.

Specimen 06 · authored component

Write the component.
Let Solix reveal the system.

One setup. Fine-grained updates. Familiar TSX. The transformation stays legible from authored source through generated templates.

counter.tsx 
import { $component, mount } from "solix";

const Counter = $component(function Counter() {
  let count = 0;
  const doubled = count * 2;

  return (
    <button onClick={() => count++}>
      Exposure {count} · index {doubled}
    </button>
  );
});

mount(Counter, document.querySelector("#app")!);
